# zaomeng

[中文](README.md) | [English](README.en.md)

`zaomeng` 不是一个泛泛的聊天机器人项目。

它更像是一套面向小说人物蒸馏、关系图谱生成、角色互动演绎的能力集合，目标很明确：

- 从小说中蒸馏人物档案
- 抽取人物关系并生成关系图谱
- 让角色按自己的人设、立场、关系和记忆开口说话

如果你想要的不是“像 AI 在陪聊”，而是“林黛玉像林黛玉，刘备像刘备，一群角色放进同一场景里还能各说各话”，这个项目就是为这件事做的。

## 你能拿它做什么

### 1. 蒸馏人物

给它一本小说，它会尽量从正文中整理出可复用的人物档案，包括但不限于：

- 性格底色
- 核心动机
- 说话风格
- 决策逻辑
- 关系倾向
- 情绪触发点
- 成长弧光

它不是只做一页“人物简介”，而是尽量产出后续还能继续拿来对话、演绎、修正的人设底稿。

项目现在支持两种蒸馏路径：

- 新建蒸馏：第一次为某个角色建立人物档案
- 增量蒸馏：如果该角色已经蒸馏过，就复用旧档案、记忆和用户修正，在新证据上继续更新

### 2. 生成人物关系图谱

除了结构化关系字段，它还会输出可视化关系图谱，方便直接看出：

- 谁信任谁
- 谁依赖谁
- 谁和谁在拉扯或对立

当前产物通常包括：

- 关系结果 Markdown
- Mermaid 源码
- HTML 图谱
- SVG 图谱

### 3. 进入角色互动

蒸馏完成后，现在是 **3 种玩法**，不是 2 种：

- `act`：你代入某个角色发言，可以单聊，也可以带着这个角色直接加入群聊
- `insert`：你不扮演书中角色，而是以“你自己”的身份进入小说场景，直接和角色互动
- `observe`：你不下场，只观察多个角色围绕一个场景自然展开

这三种模式的区别很直观：

- 想说“我来当宝玉，你们继续接”，用 `act`
- 想说“我本人走进大观园，和他们直接说话”，用 `insert`
- 想说“我先不说话，看看他们自己怎么聊”，用 `observe`

`insert` 首次进入时，会建立一张很轻的场景身份卡，通常包括：

- 角色们如何称呼你
- 你在场景里的身份
- 你偏自然闲聊、沉浸互动还是试探型互动
- 你对剧情的影响范围

## 现在的工作方式

当前版本是 **LLM-first**：

- 宿主或运行环境中的 LLM 负责真正生成语言
- `zaomeng` 负责准备 prompt、约束、人物档案、关系信息和辅助产物
- skill 路径优先复用宿主已经提供的模型能力

重点已经不是“写很多硬编码规则把句子拼起来”，而是给模型更清晰的人设和关系约束，让角色表达更像人、更像原作人物。

## 推荐上手顺序

推荐顺序很简单：

1. 提供小说文本
2. 说明你要蒸馏哪些人物
3. 等人物档案和关系图谱生成完成
4. 再进入 `act`、`insert` 或 `observe`

### 你可以直接这样说

```text
帮我从这本小说里蒸馏林黛玉、贾宝玉、薛宝钗
```

```text
蒸馏完成后，让林黛玉、贾宝玉、薛宝钗进入群聊模式
```

```text
让我扮演贾宝玉，林黛玉来回我
```

```text
让我以我自己进入红楼梦，和林黛玉、贾宝玉聊天
```

```text
生成关系图谱，我要看 HTML 和 SVG
```

## 项目结构

当前仓库大致分成三层：

- `src/`：核心源码
- `zaomeng-skill/`：可发布的 skill 包
- `tests/`：回归测试

其中 skill 包的重点资产包括：

- `prompts/`
- `references/`
- `tools/prepare_novel_excerpt.py`
- `tools/build_prompt_payload.py`
- `tools/export_relation_graph.py`

## 一句话总结

`zaomeng` 想做的，不是“会说话的 AI”。

而是让小说里的人，带着自己的性格、关系、语气和记忆，重新开口。
